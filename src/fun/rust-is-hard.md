## Why is Rust uniquely called out?

Why is Rust uniquely called out as hard, even by those who've mastered it?

I think there are a few reasons

First, it genuinely used to be

The documentation wasn't as good

There were less guides

Compiler errors weren't as useful

The tooling wasn't as complete

But that's all changed

The Rust community is full of amazing people contributing to making learning and using Rust an amazing experience

Second, I think its a bit of a badge of honour

If Rust is hard, those of us who manage to learn it must be special

I don't think anyone does this on purpose, and like I said, Rust _is_ hard

I just don't think its uniquely hard

Third, it's different

If you're coming from languages like JavaScript or Python, you now need to think about where in memory things are
stored.

If you're coming from C or C++ you need to describe how that stored data will actually be used.

...

Finally, its self perpetuating,

Everyone knows Rust is hard, because Rust is hard.

So why do I think Rust is no more difficult than any other language

## Evidence

A few years ago I had the bright idea to introduce a Rust service to an established ecosystem

The other engineers agreed the potential benefits were worth the experiment.

I built it, and our hypothesis was right,

It was significantly faster and more efficient than similar systems but...

no one wanted to touch it

No one, until we hired a junior developer straight out of bootcamp

They took to it like a duck to water

That left me wondering

This new engineer definitely found it easier to learn,

easier than the other engineers,

easier even than it'd been for me.

What was the difference

## How to master it

I'm told, by people much smarter than me

That when learning a second human language

To get really good, you have to stop translating in your head

You need to stop thinking in your first language

and start thinking in your new one

Rust works differently to JavaScript, Python, PHP, Ruby

you don't need to worry as much about whether your types are exactly right

you need to think about where things live in memory

Rust works differently to C and C++

you can stop worrying about whether you're using memory correctly

so long as you can prove it to the compiler

Start thinking like a Rust programmer

Ask for help when you get stuck

It's ok, we all get stuck

Give yourself time to learn

Be kind to yourself

And remember,

Rust is hard,

but so was your last language, and you managed that
