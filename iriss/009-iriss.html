Generics

Generics exist in many languages and are a way of creating templates that only become ‚Äúconcrete‚Äù at build time.

Functions, Structs and Enums can all be made generic by adding triangle brackets after their name, containing a list of generic parameters.

Very over you might see a single letter generic name, particularly you might see something like T as you do with Option<T>, however, you might want to hint about the purposed of the type like the Error E in Result<T, E>.

You can even use whole words, though this is seen less.

For our purposes, we‚Äôll use the letter A to represent ‚ÄúAnimal‚Äù.

Later we can fill in the Generic and make it a ‚Äúconcrete‚Äù type, in our case this will eventually be Mischievous<Cat>.

Next we‚Äôll need to update our implementations.

Implementations that involve Generic parts, need to list those after the impl to save confusing implementations on generic types and implementations on concrete variants of generic types

This just means you can implement the concrete version of Mischievous<Cat> and that‚Äôs not a generic implementation, but impl<A> Michievous<A> is.

Once we‚Äôve specified the generic parameters of the impl block, we don‚Äôt need to respecify them for each function so long as the type is being used consistently.

So, our new functions can use A for the animal parameter, and that will be linked to our state‚Äôs generic A, and our state changes can use A to show that when the state changes, the generic type will remain the same.
